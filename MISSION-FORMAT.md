# MISSION.md Format

`MISSION.md` captures why the learner is studying this topic. Every lesson, reference, and practice task should connect to it.

```md
# Mission: {Topic}

## Why

{1-3 sentences. Describe the real-world outcome, not only "understand the topic."}

## Success looks like

- {Specific observable ability}
- {Another observable ability}
- {Another observable ability}

## Constraints

- {Time, budget, safety, prior knowledge, language, learning style, or tool constraints}

## Out of scope

- {Adjacent topics to avoid for now}
```

Rules:

- Keep one mission per workspace.
- Make the mission concrete.
- If the mission is vague, ask the learner before creating a full lesson.
- Update the mission only when the learner confirms the change.
