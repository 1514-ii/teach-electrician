# Notes

## Teaching Preferences

- Korean is preferred unless the learner asks otherwise.
- Keep answers short and easy to scan.
- Start with the conclusion.
- Give one recommended next action before alternatives.
- Use simple wording when possible.
- Keep the one-problem dialogue separate from the existing HTML lesson workflow.
- In dialogue practice, ask for type, conditions, formula/reason, solution/unit, and confidence before feedback.
- Do not reveal the type or answer first. Give one staged hint at a time when the learner is stuck.
- If the setup is correct, skip arithmetic when the learner is busy.
- Diagnose concept, condition interpretation, formula choice, calculation, units, and memory retrieval after each answer.
- Summarize accuracy, hint count, confidence calibration, repeated errors, and next review target every five problems.

## Source Safety

- Do not commit paid PDFs, private work documents, personal data, or raw source archives.
- Record source names and usage notes in `RESOURCES.md` instead.
